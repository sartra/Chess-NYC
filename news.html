  • Bootstrap Grid - 3 Responsive Examples

    See how the Bootstrap grid stacks up – literally.

    Here are 3 Bootstrap examples that demonstrate different column layouts on multiple devices and screen widths. The Bootstrap grid has 4 “sizes”…

    `col-lg-*` widths >= than 1200 pixels (for desktops)

    `col-md-*` widths >= than 992 pixels (for laptops)

    `col-sm-*` widths >= than 768 pixels (for tablets)

    `col-xs-*` widths less than 768 pixels (for phones)

    You can utilize the different grid sizes to create responsive layouts that work best for each screen width or responsive “breakpoint”.

  • We’re Ready for Bootstrap 3!

    The much anticipated 3.0 release of Bootstrap is due out today, Aug. 19th.  

    Bootstrap creators Mark Otto and Jacob Thorton have re-considered everything to improve Bootstrap 3. The most popular responsive framework is now lighter-weight, simpler to use and easier to customize. “Everything has changed”, according to the official Bootstrap blog. “With over ~1,600 commits, ~72,000 additions/deletions, and ~300 files changed…”. “Mobile First” is the keystone of the Bootstrap 3 release, making it better than ever as a foundation for designing sites that work on multiple devices.

    Improvements

    The 3.0 standout features are 1) the more responsive grid, and 2) easier customization. Since the new version has 4 grid sizes, developers can quickly design for multiple devices (deskop, laptop, tablet, phones) without having to write custom CSS media queries or special  HTML markup. The new grid is really powerful. Also, the Bootstrap elements have a cleaner “flat” look.

    You can read more about the Bootstrap 3 improvements here.

    Impact

    According to meanpath, 1 percent of the 150 million websites in its index now use Bootstrap. That means potentially 1000’s of Web sites and developers considering the migration path to 3, and backwards compatibility.

    An official migration section was just added to the Bootstrap docs, which provides a helpful summary of the class changes from 2.x to 3.0. There are also several upgrade tools that have emerged:

    • Bootstrap 3 upgrade service
    • Divshot Bootstrap upgrader
    • Bootstrap migrator
    • Bootstraptor upgrade service

    At Bootply, we have also integrated the upgrade tool directly into the code editor, so that you can easily convert 2.x Bootstrap HTML code with the click of a button. The editor UI has also been updated with the new Bootstrap 3 grid classes, and we’ll be adding more Bootstrap 3 snippets and examples shortly.

  • What’s New in Bootstrap 3
    In case you haven’t been following the Bootstrap 3 work-in-progress, let’s start with a quick summary of what’s changed for 3. Everything has changed. For starters, the 2.x grid has been completely rethought in an effort to acheive “mobile first”. From Bootstrap 2.3.2 to 3.0 RC 1 there were over 1,600 commits, 72,000 additions/deletions and 300 changed files.

    Mobile First

    “Mobile first” is a responsive Web design practice that prioritizes consideration of smart phones and mobile devices when creating Web pages. For Bootstrap 3, there is no longer a separate bootstrap-responsive.css file. All responsive features are now compiled into a single core bootstrap.cssfile. Bootstrap 3 allows Web page elements (text, images, buttons, paragraphs, etc..) to scale accordingly as the viewing area, size and resolution changes from wide-screen desktop to tablet to small screen smartphone. Mobile-first elimates to need to make separate desktop and mobile versions of your Web site.


    The New Grid

    The more powerful (and 100% fluid) Bootstrap 3 grid now comes in 3 sizes. Tiny (for smartphones .col-*), Small (for tablets .col-sm-*) and Large (for laptops/desktops .col-lg-*). The 3 grid sizes enable you to control grid behavior on different devices (desktop, tablet, smartphone, etc..).

    Unlike 2.x, Bootstrap 3 does not provide a fixed (pixel-based) grid. While a fixed-width layout can still be acheived using a simple custom wrapper, there is now only one percentage-based (fluid) grid. The .container and .row classes are now fluid by default, so don’t use .row-fluid or .container-fluid anymore in your 3.x markup.
